How does the Web works

To access the WWW or a local website a web client is used. The client retrieves content from resources that are stored on the webserver which is a computer that accepts HTTP and HTTPS requests and then graphically renders the requested page or sends an error message.

URL

A Uniform Resource Locator (URL), colloquially termed a web address, is a reference to a web resource that specifies its location on a computer network and a mechanism for retrieving it. The format combines the scheme followed by :// and the pre-existing system of domain names with file path syntax, where slashes are used to separate directory and filenames.

The general components of an URL

hypertext Transfer control (http/s); ://; World wide web (www); Domain name, Top level domain; Port number; path; (?) query; (#) fragment;

Web Server

A web servers is a computer software and underlying hardware that accepts requests via HTTP or HTTPS. The web server connects to the internet and supports physical data interchange with other devices connected to the web. On the hardware the web server software and website files will be stored. The contend in webpages are either static or dynamic. Known Application Server: Nginx, Apache

    HTML (Hypertext Markup Language) is used to write the structure of a webpage (skeleton). CSS (Cascading Style Sheet) can be used to make the website look nicer. You can format text, position items and more. The <html> element is the root of an html document. It contains all other elements except for the <!DOCTYPE html element> .
